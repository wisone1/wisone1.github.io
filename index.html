Debian 设置root用户登陆

Debain系统默认是不允许root账户直接登陆的，通过下面方式可以设置.

1、首先修改gdm3的设定文件(/etc/gdm3/daemon.conf)，在[security]字段后面追加如下一行：

 [security]
  AllowRoot = true

2、最后修改gdm3的登录pam文件：/etc/pam.d/gdm-password

  将auth required pam_succeed_if.so user != root quiet_success注释掉 //本行前加#

iptables简单设置

iptables简单学习笔记

iptables是一款基于命令行的防火墙策略管理工具，有大量的参数，学习难度较大，比如”四表五链”等概念，这里只学习基本常用参数并搭配使用。

iptables属于数据链路层服务，所以可以根据流量的源地址，目的地址，传输协议，服务类型等信息进行匹配。防火墙策略归的匹配顺序为从上至下，执行优先级亦是。

一.常用参数即作用：

二.实验

1）查看已有的防火墙规则链
iptables -L

  1. 清空已有的防火墙规则链
    iptables -F

3)把INPUT规则链的默认策略设置为拒绝
iptables -P ACCETP DROP
注意：规则链的默认策略拒绝动作只能是DROP,而不能是REJECT

  1. 向INPUT链中添加允许ICMP流量进入的策略规则（添加一条ICMP流量进入规则，允许ping命令检测行为
    iptables -I INPUT -p icmp -j ACCEPT

  2. 删除INPUT规则链中4)中刚刚加入的那条策略，并把默认策略设置为允许
    iptables -D INPUT 1 #我的INPUT策略中该策略为第一条
    iptables -P INPUT ACCEPT

  3. 将INPUT规则链设置为只允许制定网段的主机访问本级的22端口，拒绝来自其他所有主机的流量（可以直接写IP地址，如对网段匹配则需要写为子网掩码的形式
    iptables -I INPUT -s 192.168.10.0/24 -p tcp –dport 22 -j ACCEPT
    iptables -A INPUT -p tcp –dport 22 j REJECT
    注意：防火墙的策略规则是从上到下顺序匹配的，所以一定要把允许动作放在拒绝动作之前！！！

  4. 向INPUT规则链中添加拒绝所有人访问本级12345端口的策略规则
    iptables -I INPUT -p tcp –dport 12345 -j REJECT
    iptables -I INPUT -p udp –dport 12345 -j REJECT

  5. 向INPUT规则链中添加拒绝192.168.10.5主机访问本机80端口的策略规则
    iptables -I INPUT -p tcp -s 192.168.10.5 –dport 80 -j REJECT

  6. 向INPUT规则链中添加拒绝所有主机访问本机1000-1024端口的策略规则
    iptables -A INPUT -p tcp –dport 1000:1024 -j REJECT
    iptables -A INPUT -p udp –dport 1000:1024 -j REJECT

特别注意：使用iptables命令配置防火墙规则默认会在系统下一次重启时失效，如果想让配置的防火墙策略永久生效，还要执行保存命令： iptables-save

apt-get 与 dpkg 参数

apt dpkg

Common apt commands

apt-get install Downloads and all of its dependencies(依赖), and installs or upgrades(升级) them.

apt-get remove [–purge] Removes and any packages that depend on it. –purge specifies(指定/规定/说明) that packages should be purged.

apt-get update Updates packages listings from the repo, should be run at least once a week.

apt-get upgrade Upgrades all currently installed packages with those updates available from the repo. should be run once a week.

apt-get dist-upgrade [-u] Similar to apt-get upgrade, except that dist-upgrade will install or remove packages to satisfy dependencies(满足依赖关系).

apt-cache search Searches packages and descriptions for .

apt-cache show Shows the full description of .

apt-cache showpkg Shows a lot more detail(细节) about , and its relationships(关系) to other packages.

man apt Will give you more info on these commands as well as many that are in less common usage.

Common dpkg commands

dpkg -i <package.deb> Installs a package file; one that you downloaded manually(手动), for example.

dpkg -c <package.deb> Lists the contents of <package.deb> a .deb file.

dpkg -I <package.deb> Extracts(提取) package information from <package.deb> a .deb file.

dpkg -r Removes an installed package named

dpkg -P Purges(清除) an installed package named . The difference between remove and purge is that while remove only deletes data and executables(可执行文件), purge also deletes all configuration files in addition.

dpkg -L Gives a listing of all the files installed by . See also dpkg -c for checking the contents of a .deb file.

dpkg -s Shows information on the installed package . See also apt-cache show for viewing package information in the Debian archive and dpkg -I for viewing package information extracted from a .deb file.

dpkg-reconfigure Reconfigures an installed package

man dpkg Will give you more info on these commands as well as many that are in less common usage.

1.创建仓库

2.使用Git以及配置SSH

在这里我们需要对SSH进行配置，我们打开git bash或者直接打开终端。
git config –global user.name “这里改为你的github用户名”
git config –global user.email “这里改为你注册github用的邮箱”
你可以通过输入以下命令确认有没有输入错误，
git config user.name && git config user.email
然后我们创建一个SSH公钥用于和github的SSH连接，在终端输入：
ssh-keygen -t rsa -C “这里改为你注册github用的邮箱”
它会提示你在.ssh文件夹里生成了SSH公钥，我们只需要cd进.ssh文件夹里 然后用cat命令查看并复制下id_rsa.pub里的公钥内容
我们回到github，然后点击头像再点击setting，找到SSH and GPG keys的选项，新添加一个SSH key，把刚才复制的内容粘贴进去，然后点一下Add SSH key之类的选项就能添加了。
回到我们的bash输入ssh -T git@github.com测试一下是否成功，成功的话就可以进行下一步了。

Debian 清除已删除软件包的配置文件

命令如下：

用途：删除已经删除的软件包的残留配置文件。因为debian系统中remove包不删除配置文件，而purge可以删除，所以安装卸载的软件多了，可以用这个清理残留配置文件。

说明：
1） dpkg -l
列出系统中安装的所有包的状态，’ii’开头的是正常安装的包，’rc’ 开头的是软件删除留下的配置文件包，其他状态则是有错误的状态。
2） grep “^rc”
提取以rc开头的包，即提取有残留配置文件的包的信息行。
3） awk ‘{print $2}’
打印这些包的名字，他们位于 dpkg -l 输出结果的第二个字段。
4） xargs aptitude -y purge
把第三部的输出放在 aptitude -y purge 后面，purge命令清除配置文件， -y参数自动对后面的 aptitude命令的提示回答是，这个参数是很危险的，一般情况下不要使用，因为在这个地方我们确定删除这些残留文件没有问题，所以才使用。

也可以用 purge 参数 移除软件包关联的所有数据，如用 dpkg -P 软件包，apt-get remove –purge 软件包 或者 aptitude purge 软件包 彻底删除软件和关联所有数据，移除这种数据，一定要慎之又慎。
