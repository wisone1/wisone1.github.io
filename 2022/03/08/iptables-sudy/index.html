iptables简单学习笔记

iptables是一款基于命令行的防火墙策略管理工具，有大量的参数，学习难度较大，比如”四表五链”等概念，这里只学习基本常用参数并搭配使用。

iptables属于数据链路层服务，所以可以根据流量的源地址，目的地址，传输协议，服务类型等信息进行匹配。防火墙策略归的匹配顺序为从上至下，执行优先级亦是。

二.实验

  1. 查看已有的防火墙规则链
    iptables -L

  2. 清空已有的防火墙规则链
    iptables -F

  3. 把INPUT规则链的默认策略设置为拒绝
    iptables -P ACCETP DROP
    注意：规则链的默认策略拒绝动作只能是DROP,而不能是REJECT

  4. 向INPUT链中添加允许ICMP流量进入的策略规则（添加一条ICMP流量进入规则，允许ping命令检测行为
    iptables -I INPUT -p icmp -j ACCEPT

  5. 删除INPUT规则链中4)中刚刚加入的那条策略，并把默认策略设置为允许
    iptables -D INPUT 1 #我的INPUT策略中该策略为第一条
    iptables -P INPUT ACCEPT

  6. 将INPUT规则链设置为只允许制定网段的主机访问本级的22端口，拒绝来自其他所有主机的流量（可以直接写IP地址，如对网段匹配则需要写为子网掩码的形式
    iptables -I INPUT -s 192.168.10.0/24 -p tcp –dport 22 -j ACCEPT
    iptables -A INPUT -p tcp –dport 22 j REJECT
    注意：防火墙的策略规则是从上到下顺序匹配的，所以一定要把允许动作放在拒绝动作之前！！！

  7. 向INPUT规则链中添加拒绝所有人访问本级12345端口的策略规则
    iptables -I INPUT -p tcp –dport 12345 -j REJECT
    iptables -I INPUT -p udp –dport 12345 -j REJECT

  8. 向INPUT规则链中添加拒绝192.168.10.5主机访问本机80端口的策略规则
    iptables -I INPUT -p tcp -s 192.168.10.5 –dport 80 -j REJECT

  9. 向INPUT规则链中添加拒绝所有主机访问本机1000-1024端口的策略规则
    iptables -A INPUT -p tcp –dport 1000:1024 -j REJECT
    iptables -A INPUT -p udp –dport 1000:1024 -j REJECT

特别注意：使用iptables命令配置防火墙规则默认会在系统下一次重启时失效，如果想让配置的防火墙策略永久生效，还要执行保存命令： iptables-save
